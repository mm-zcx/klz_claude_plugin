## Qt 编译错误诊断

### 示例 1：MOC 错误

**问题**：
```
undefined reference to `vtable for MyClass`
```

**解决方式**：
直接向 Claude 描述问题：
```
我遇到了一个 Qt 编译错误：
undefined reference to `vtable for MyClass'

我的类定义在 myclass.h 中，使用了 Q_OBJECT 宏。
```

### 示例 2：信号槽连接错误

**问题**：信号槽连接失败

**解决方式**：
```
Qt 信号槽连接有问题，帮我检查：
QObject::connect(sender, &Sender::signalName, receiver, &Receiver::slotName);

编译器提示没有匹配的函数。
```

### 示例 3：CMake 配置问题

**问题**：CMake 找不到 Qt 包

**解决方式**：
```
我在使用 CMake 构建 Qt 项目时遇到错误：
Could not find Qt6

我的 CMakeLists.txt 配置了 find_package(Qt6 REQUIRED)
```

## Gitea 集成

### 示例 1：创建 Issue

**请求**：
```
帮我在 Gitea 上创建一个 Issue：

仓库：myproject
标题：修复登录页面样式问题
描述：
在 Chrome 浏览器中，登录页面的按钮对齐有问题。
截图已附上。

优先级：高
标签：bug, ui
```

### 示例 2：创建 Pull Request

**请求**：
```
创建一个 Gitea Pull Request：

源分支：feature/new-api
目标分支：main
标题：新增用户 API 接口
描述：
- 新增用户查询接口
- 添加单元测试
- 更新 API 文档

审查者：@team-leads
```

### 示例 3：查询仓库信息

**请求**：
```
查询 Gitea 仓库信息：
仓库：myproject
分支：main

需要获取：
- 最近的提交记录
- 开放的 Issue
- 活跃的 Pull Request
```

### 示例 4：发布管理

**请求**：
```
在 Gitea 上创建新发布：

仓库：myproject
标签：v1.2.0
标题：版本 1.2.0
描述：
## 新功能
- 用户认证
- API 接口

## Bug 修复
- 修复登录问题
- 优化性能

附件：myproject-v1.2.0.zip
```

## 技巧提示

### 1. 提供上下文
- 说明 Qt 版本（Qt 5 或 Qt 6）
- 描述项目类型（QWidget、QML、控制台等）
- 提供相关代码片段

### 2. 明确需求
- 描述具体问题或目标
- 指定关注的方面（性能、规范、安全等）
- 提供约束条件（编译器、平台等）

### 3. 迭代改进
- 根据建议进行修改
- 再次请求审查或验证
- 记录解决方案

### 4. 文档和注释
- 请求添加中文注释
- 生成使用文档
- 创建示例代码
